Overview of the UI

Navbar

The navbar has 8 icons. Each of the description is shown below:

  1. Undo: The undo button will go back in history of what was written in the coding text area.
  2. Redo: The redo button is going to redo any of the items that was undo.
  3. File Upload: This button focuses on uploading any (and only) text files found in your computer. The file will show up in the coding text area.
  4. New File: This file will clear out all of the items shown in the coding text area, tabs sections, and in the notifications section.
  5. Save File: The Save File button will save any of the current information that is in the coding text area.
  6. Compile: Compile button will only check if there is syntax or lexical mistakes found throughout your code.
  7. Run: Run button will run the program. Ensure that is was previously compiled.
  8. Tutorial: The tutorial button is data that will not only explain how the UI works but also explain more about the PL/0 language.

Coding Text Area

The coding text area is to the left hand side of the UI. Note that the reserved words in the language and numbers will be a different color than the rest.

Tabs

There are five tabs shown in the right hand side of the page:

  1. Lexical Table: The Lexical Table is created by the Lexical Machine. It contains all of the symbols with its corresponding number that its represented in the PL/0 language.
  2. Tokens: The tokens are created by the Lexical Machine. The tokens will be used for the Syntax Analyzer Machine.
  3. Mcode: The mcode is generated by the Syntax Analyzer. It will be used by the virtual machine to run the code.
  4. Stacks & Registers: When the mcode data is passed and run by the Virtual Machine, its starts generating stacks and assigns a few registers. The final result is shown in this tab.
  5. Stack Review: The Stack Review section is suppose to help students understand how the stack will look for the code.

Notifications

The Notification is at the bottom of the page. It will notify the users if there is any errors in the code or if everything is synatically correct.
